## 2. 职位选择策略 (Job Selection Strategy)

职位选择是 **"选择大于努力"** 的博弈。
**本科学历分析**: 在电子信息工程领域，本科学历在 **市直/城区级单位** 仍有大量机会，但在 **市级高校/科研院所** 会受到限制（通常要求硕士）。

### 2.1 选岗核心算法 (Selection Algorithm)

```go
func SelectJob(jobs []Job) Job {
    // 优先级排序：专业匹配 > 学历匹配 > 竞争比
    
    // Constraint: 学历过滤器
    validJobs := FilterByDegree(jobs, "本科")
    
    // Strategy 1: 市直机关技术岗 (推荐)
    // 很多市局下属的信息中心、监测中心明确招录本科
    techJobs := FilterByUnitType(validJobs, "市直机关下属事业单位")
    
    // Strategy 2: 城区/开发区岗位 (高性价比)
    // 青秀区、高新区等大数据局，待遇好且招本科
    districtJobs := FilterByArea(validJobs, "城区级")
    
    // Strategy 3: 避坑逻辑 (Avoid Pitfalls)
    // 1. 避开要求 "硕士研究生" 的市属高校教师岗
    // 2. 避开 "三不限" (竞争过大)
    
    return OptimizeChoice(techJobs, districtJobs)
}
```

### 2.2 电子信息工程推荐报考方向 (Recommended Positions) - **本科版**

#### A. 核心技术岗 (Core Tech) - **本科友好区**
*   **单位类型**: 
    *   **市/城区大数据发展局**: 核心业务岗，通常招本科。
    *   **市/城区融媒体中心**: 技术维护岗，本科起步。
    *   **公安局网安/技侦**: 警务技术岗（需体能测试），本科为主。
    *   **应急管理局**: 应急指挥中心技术岗。
*   **优势**: 招聘人数多，专业限制严格（卡掉文科生），本科即可报考。

#### B. 垂管与垂直系统 (Vertical System)
*   **单位类型**: 
    *   **生态环境监测中心**: 部分运维岗招本科。
    *   **无线电监测中心**: 核心对口单位，待遇优厚。
*   **注意**: 部分省级垂管单位可能要求硕士，需仔细甄别。

#### C. 职业院校 (Vocational Schools) - **需谨慎**
*   **现状**: 南宁职业技术学院等市属高校，专任教师多要求硕士。
*   **本科机会**: 
    *   **实训指导教师**: 部分岗位放宽至本科+技能证书。
    *   **行政/辅导员**: 竞争极其激烈，不建议作为首选。
    *   **中职学校**: 如南宁市第一职业技术学校，专业课教师本科有机会。

## 3. 考试架构分析 (Exam Architecture)

### 3.1 考试场次 (Exam Sessions)

广西事业单位统考采用 **"一天两场，连续作战"** 模式：

```go
type ExamSchedule struct {
    Date        string // "2026-03-30" (预计)
    Session1    Exam   // 职业能力倾向测验
    Session2    Exam   // 综合应用能力
    TotalTime   int    // 总时长：约3.5小时（含中间休息）
}

// 考试时间表
var schedule = ExamSchedule{
    Session1: Exam{
        Name:     "职业能力倾向测验",
        Start:    "08:30",
        End:      "10:00",
        Duration: 90, // 分钟
    },
    Session2: Exam{
        Name:     "综合应用能力",
        Start:    "10:00", // 注意：中间可能有10分钟休息
        End:      "12:00",
        Duration: 120, // 分钟
    },
}
```

**关键点**：
- **笔试阶段**：仅此一天，两科连考。没有补考机会，必须一次性通过。
- **面试阶段**：笔试通过后，由各招聘单位另行通知（通常在笔试后1-2个月）。

### 3.2 题型深度解析 (Question Type Analysis)

#### 3.2.1 职业能力倾向测验 (Job Aptitude Test)

**总题量**: 100题 | **总时长**: 90分钟 | **平均每题**: 54秒

这是一个 **高并发、低延迟** 的测试系统，要求你在极短时间内做出正确判断。

| 模块 | 题量 | 建议时间 | 难度 | 策略 |
|------|------|----------|------|------|
| **常识判断** | 20题 | 8分钟 | ⭐⭐⭐ | `Cache` 策略：秒选，不纠结 |
| **言语理解与表达** | 25题 | 32分钟 | ⭐⭐ | `Parser` 策略：找中心句、关键词 |
| **数量关系** | 10题 | 12分钟 | ⭐⭐⭐⭐ | `Fallback` 策略：只做简单题 |
| **判断推理** | 30题 | 35分钟 | ⭐⭐⭐ | `Rule Engine` 策略：套公式 |
| **资料分析** | 15题 | 23分钟 | ⭐⭐ | `Math Library` 策略：必须拿满分 |

**详细题型拆解**：

1. **常识判断 (20题, 8分钟)**
   - **题型**: 单选
   - **内容**: 政治、法律、经济、科技、人文、地理、广西区情
   - **Go思想**: 这是 `Cache Miss` 率最高的模块。不要花时间专门学，利用碎片时间刷题混眼熟即可。

2. **言语理解与表达 (25题, 32分钟)**
   - **题型**: 
     - 片段阅读（15题）：主旨概括、意图推断、细节理解
     - 逻辑填空（10题）：实词、成语、关联词
   - **Go思想**: 像 `Parser` 一样，快速识别句子结构（主谓宾），找到中心句和转折词。

3. **数量关系 (5题, 12分钟)**
   - **题型**: 数学运算（工程问题、行程问题、容斥原理、排列组合）
   - **Go思想**: 这是 `CPU密集型` 任务。在职考生时间有限，**战略性放弃**。只做前5道简单题，后5道直接蒙。

4. **判断推理 (35题, 35分钟)**
   - **题型**:
     - 图形推理（5题）：数量、位置、样式、属性
     - 定义判断（10题）：抓关键词
     - 类比推理（10题）：找关系
     - 逻辑判断（5题）：假言命题、联言选言
   - **Go思想**: 像 `Rule Engine` 一样，每个题型都有固定套路。图形推理看"点线面角素"，逻辑判断套"肯前必肯后，否后必否前"。

5. **资料分析 (15题, 23分钟)**
   - **题型**: 4篇材料，每篇3-4题
   - **内容**: 增长率、增长量、比重、平均数、倍数
   - **Go思想**: 这是 **必须拿满分** 的模块。像优化 `Math Library` 一样，死磕速算技巧（截位直除、首数法、特征数字法）。

---

#### 3.2.2 综合应用能力 (Composite Application Ability)

**总时长**: 120分钟 | **题量**: 3-4题（主观题）

这是一个 **CPU密集型 + IO密集型** 的混合任务，需要深度思考和逻辑输出。

**分类别题型**（根据岗位类别ABCDE）：

| 类别 | 适用岗位 | 核心题型 | 题量 |
|------|----------|----------|------|
| **A类** | 综合管理 | 案例分析、公文写作、应急处理 | 4题 |
| **B类** | 社科专技 | 概念分析、校阅改错、论证评价、材料作文 | 4题 |
| **C类** | 自科专技 | 科技文献阅读、科技实务、材料作文 | 3题 |
| **D类** | 中小学教师 | 辨析题、案例分析、教学设计 | 3题 |
| **E类** | 医疗卫生 | 医学基础知识、案例分析、实务题 | 2题 |

**电子信息工程岗位通常对应 A类（综合管理）或 C类（自然科学专技）**：
*   **报考行政/管理岗 (如信息中心)** -> **A类** (考概率最大)
*   **报考科研/技术岗 (如气象局)** -> **C类** (考概率较小，但也存在)
*   **Action**: 务必在报名时查看职位表的“考试类别”一栏。

**通用解题算法**：

```go
func SolveCompositeQuestion(material Material, question Question) Answer {
    // Step 1: Input (审题) - 最关键
    intent := ParseQuestion(question) // 识别题目意图
    scope := LocateMaterial(material, intent) // 锁定材料范围
    
    // Step 2: Process (找点)
    keyPoints := ExtractKeyPoints(scope, intent) // 提取核心信息
    // 关键词：原因、影响、对策、问题、意义
    
    // Step 3: Output (加工)
    answer := FormatAnswer(keyPoints) // 结构化输出
    // 必须分条列点：1. 2. 3.
    // 使用官方语言："加强宣传" -> "构建多维宣传矩阵"
    
    return answer
}
```

**时间分配建议**（以A类为例）：
- **审题+读材料**: 20分钟
- **第1题（案例分析）**: 25分钟
- **第2题（案例分析）**: 25分钟
- **第3题（应急处理）**: 25分钟
- **第4题（公文写作）**: 30分钟
- **检查**: 5分钟

## 4. 核心架构：在职流水线 (Pipeline)

### Stage 1: 极简重构 (Refactoring) - 2025.12 (30天)
> **原则**: `KISS` (Keep It Simple, Stupid)

*   **痛点**: 下班后不想看书。
*   **策略**:
    *   **网课倍速**: 只看核心理论，跳过废话。使用 1.5x 或 2.0x 倍速。
    *   **放弃非核心**: 常识判断不要专门花时间学，仅靠碎片时间刷题。
    *   **抓大放小**: 重点攻克 **资料分析** 和 **言语理解**，这两项决定了职测的下限。

### Stage 2: 碎片化集成 (Integration) - 2026.01 (30天)
> **原则**: `MapReduce` (分布式处理)

*   **痛点**: 没时间做整套卷子。
*   **策略**:
    *   **Map (拆分)**: 将一套试卷拆分为 5 个模块。
    *   **Reduce (汇总)**: 利用通勤、午休分别完成模块，晚上汇总复盘。
    *   **综应积累**: 每天利用碎片时间背 1-2 个综应金句或框架，不要指望考前突击。

### Stage 3: 压力测试 (Load Testing) - 2026.02 (28天)
> **原则**: `Simulate Production` (全真模拟)

*   **痛点**: 只有周末有空。
*   **策略**:
    *   **周末模考**: 周六上午必须严格按考试时间（8:30-12:00）进行全真模拟，雷打不动。这是你适应考试节奏的唯一机会。
    *   **周日复盘**: 周日专门用来分析周六的错题，并进行针对性补强。

## 5. 每日 Goroutine 调度 (关键)

在职备考的精髓在于 **时间分片 (Time Slicing)**。我们将一天分为 4 个并发协程：

### G1: Early Morning Worker (晨间协程) - `06:30 - 07:30`
> **Priority**: `High` | **Context**: `Background`

*   **状态**: 大脑最清醒，无工作打扰。
*   **任务**: **CPU密集型任务**。
    *   做 2 篇言语理解 / 1 组资料分析。
    *   背诵综应的核心考点。
*   **为何早起**: 晚上下班后的时间是不可控的（加班、聚餐、疲惫），只有早上的时间完全属于你。

### G1.5: Morning Buffer (晨间缓冲) - `07:30 - 08:30`
> **Priority**: `Medium` | **Context**: `Life Support`

*   **07:30 - 08:00 (Life)**: 营养早餐 & 洗漱。身体是革命的本钱，低血糖会直接导致 G2/G3 宕机。
*   **08:00 - 08:30 (Light IO)**: **晨读/预热**。
    *   利用早饭后的碎片时间，进行低强度的记忆工作。
    *   内容：成语积累、综应金句朗读、错题本快速浏览。

### G2: Commute Microservice (通勤微服务) - `08:30 - 09:00`
> **Priority**: `Low` | **Context**: `Mobile`

*   **状态**: 公交/地铁/开车，环境嘈杂。
*   **任务**: **IO等待型任务**。
    *   **听觉**: 听时政热点磨耳朵（推荐：半月谈、人民日报）。
    *   **视觉**: 手机刷常识题，刷完即忘，混个眼熟。

### G3: Lunch Break Worker (午休协程) - `12:00 - 13:30`
> **Priority**: `Medium` | **Context**: `Office`

*   **状态**: 需要快速切换状态。
*   **任务序列**: `Motion -> Study -> Eat -> Sleep`。
    *   **12:00 - 12:20 (Motion)**: 简单运动（拉伸/散步），激活身体，摆脱上午工作的疲惫。
    *   **12:20 - 12:40 (Study)**: 刷 15 道图形推理 / 错题浏览（利用运动后的兴奋期）。
    *   **12:40 - 13:00 (Eat)**: 快速进食，补充能量。
    *   **13:00 - 13:30 (Sleep)**: **强制午睡**。即使睡不着也要闭目养神，保证下午和 G4 的电量。

### G4: Night Batch Job (晚间批处理) - `20:00 - 23:00`
> **Priority**: `Critical` | **Context**: `Deep Work`

*   **状态**: 容易疲惫，需对抗惰性。
*   **任务**: **系统性学习**。
    *   **20:00 - 21:00**: 听网课/学新知识点。
    *   **21:00 - 22:00**: 专项刷题训练。
    *   **22:00 - 22:30**: 综应写作练习（哪怕只写提纲）。
    *   **22:30 - 23:00**: **洗澡/放松/早睡**（保证 G1 重启成功）。

## 6. 异常处理与熔断机制 (Circuit Breaker)

在职备考最怕的是“因为加班/出差导致计划中断，进而自暴自弃”。

```go
func HandleWorkOverload(workPressure int) {
    if workPressure > Threshold {
        // 触发熔断
        CircuitBreaker.Open()
        // 降级策略：只保留最低限度的维持
        KeepAlive() // 仅刷10道题，维持手感
        log.Println("Switching to maintenance mode")
        return
    }
    // 恢复正常备考
    ContinueStudy()
}
```

*   **Context Cancellation**: 如果某天必须加班到很晚，**直接取消当晚复习计划**，回家睡觉。不要熬夜补进度，否则会引发连锁崩溃。
*   **Keep Alive**: 即使最忙的日子，也要在睡前看 5 分钟错题，保持连接心跳。

## 7. 效率工具 (Optimization Tools)

1.  **降噪耳机**: 在职党神器，物理屏蔽环境噪音，快速进入 `Flow` 状态。
2.  **专注App**: Forest / 番茄土豆。将手机锁死，防止 `Interrupt` (微信/抖音) 抢占 CPU。
3.  **平板/电子书**: 方便随身携带，利用会议间隙 `Read Only`。
