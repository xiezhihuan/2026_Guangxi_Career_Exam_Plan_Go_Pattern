# 核心考点总结：截位直除法 (Direct Division)

> **关联笔记**: [Day 1 资料分析公式](../notes/Day1_DataAnalysis_Formula.md)

## 1. 算法核心 (Algorithm Core)

在资料分析中，**截位直除**是解决计算问题的“标准库”函数。其核心思想是**按需计算**，在**精度 (Precision)** 和 **时间 (Latency)** 之间做权衡。

*   **输入**: 分子 (Numerator), 分母 (Denominator), 选项 (Options)
*   **输出**: 最接近的选项
*   **原则**: 永远只处理分母（分母对误差影响大），分子保持不变或微调。

## 2. 策略选择 (Strategy Pattern)

根据**选项差距 (Gap)** 动态选择截位策略：

### 策略 A：分母截两位 (High Speed)
*   **触发条件**: 选项首位不同，或选项差距明显 (> 10%)。
*   **操作**: 分母四舍五入保留前 **2** 位有效数字。
*   **真题示例**:
    > 题目: $4829 \div 1.23 = ?$
    > 选项: A. 3521  B. 3926  C. 4215  D. 4520
    *   **分析**: 选项差距很大 (35, 39, 42, 45)，首位或次位明显不同。
    *   **截位**: 分母 1.23 $\rightarrow$ 1.2。
    *   **计算**: $48 / 1.2 = 40$。
    *   **决策**: 只有 B 选项 (3926) 最接近 4000。选 B。

### 策略 B：分母截三位 (High Precision)
*   **触发条件**: 选项首位相同，且次位非常接近（差距 < 10%）。
*   **操作**: 分母四舍五入保留前 **3** 位有效数字。
*   **真题示例**:
    > 题目: $54321 \div 1.234 = ?$
    > 选项: A. 43850  B. 44020  C. 44150  D. 44320
    *   **分析**: 选项前两位都是 43/44，差距极小 (<1%)。
    *   **截位**: 分母 1.234 $\rightarrow$ 123。
    *   **计算**: $54300 \div 123 \approx 441...$。
    *   **决策**: 选最接近的 C。

## 3. 边界情况处理 (Edge Cases)

### 3.1 中间数 x.5
*   当分母截位位数为 5 时（如 1.5, 1.50），建议稍微多看一位或直接计算，因为 5 处于中间，误差可能被放大。
    *   例: $123 / 155$。若截两位为 $123/16$，误差可能偏大；建议看作 $123/15.5$ 或直接约分。

### 3.2 数量级 (Orders of Magnitude)
*   **忽略小数点**: 考试中通常选项的数量级相同（如 A. 123  B. 134），此时计算可以直接忽略小数点，只算有效数字（$12345 / 567$ 可看作 $123 / 57$）。
*   **注意单位**: 如果选项数量级不同（A. 1.2%  B. 12%），则必须严格保留小数点或估算量级。

## 4. 实战代码 (Pseudo-Code)

```go
// 模拟考试中的决策过程
func Solve(numerator, denominator float64, options []float64) float64 {
    gap := CalculateGap(options)
    
    var d float64
    if gap == Large {
        // 策略 A: 追求速度
        d = Round(denominator, 2) 
    } else {
        // 策略 B: 追求精度
        d = Round(denominator, 3) 
    }
    
    result := numerator / d
    return FindClosest(result, options)
}
```
