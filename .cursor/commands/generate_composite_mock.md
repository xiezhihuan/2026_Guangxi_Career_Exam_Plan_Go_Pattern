# 命令：/generate_composite_mock

本规则定义了如何生成 **综应A类模拟材料（无答案版本）**，用于日常练习。生成的材料包含题目、材料、练习任务和答题区，但不包含参考答案（参考答案需通过 `/answer_reference` 命令生成）。

## 触发方式

用户输入：`/generate_composite_mock` 或 `/generate_composite_mock [主题关键词]`

## 核心逻辑：模拟材料生成策略 (Mock Material Generation Strategy)

生成模拟材料时，必须遵循以下原则：

1. **题型多样性**: 涵盖综应A类常见题型：
   - 概括归纳题（问题、原因、影响等）
   - 提出对策题（解决问题、改进措施等）
   - 应急处理题（突发事件应对、舆情处理等）
   - 公文写作题（通知、报告、方案等）

2. **材料真实性**: 
   - 材料应贴近实际工作场景（社区治理、公共服务、突发事件等）
   - 问题描述具体、有层次（通常包含3-4个问题维度）
   - 包含具体数据、案例、人物对话等细节

3. **难度适中**: 
   - 材料长度控制在500-800字
   - 问题层次清晰，便于梳理
   - 符合事业编考试难度标准

4. **格式规范**: 
   - 严格按照综应A类考试格式
   - 包含明确的作答要求和字数限制
   - 提供答题提示，帮助用户规范作答

## 执行步骤 (Execution Pipeline)

1. **确定主题和题型**
   *   如果用户提供了主题关键词，优先使用该主题
   *   如果未提供，从常见主题库中随机选择：
     - 社区治理类：停车管理、垃圾分类、老旧小区改造、物业管理等
     - 公共服务类：政务服务、教育医疗、文化体育、社会保障等
     - 应急管理类：突发事件应对、舆情处理、安全生产等
     - 乡村振兴类：产业发展、环境整治、乡风文明等
   *   根据主题确定题型（优先选择"提出对策题"，这是综应A类高频题型）

2. **生成材料内容**
   *   **背景介绍**: 1-2段，介绍基本情况（时间、地点、主体、规模等）
   *   **问题描述**: 3-4个问题维度，每个问题包含：
     - 问题标题（如"问题一：XXX"）
     - 问题表现（具体现象、数据、案例）
     - 问题影响（如有）
   *   **补充信息**: 1段，介绍已尝试的解决措施或面临的困难

3. **生成练习任务**
   *   **题目**: 明确角色定位（如"假如你是XX社区居委会的工作人员"）
   *   **作答要求**: 包含5个方面：
     1. 对策具体可行，具有针对性
     2. 分条列点，格式规范（使用"一、二、三、四"或"首先、其次、再次、最后"等结构）
     3. 语言符合政府公文写作规范，避免口语化表达
     4. 字数控制在300-400字（根据题型调整）
     5. 建议用时：40分钟（含阅读材料时间）

4. **生成答题区**
   *   **第一步：问题梳理**（可在草稿纸上完成）
     - 提供4个问题空位，供用户填写
   *   **第二步：对策提炼**（分条列点作答）
     - 空白作答区域

5. **生成答题提示**
   *   **格式要求**: 强调使用规范的结构词
   *   **对策维度**: 提示可以从哪些角度思考（规划建设、管理制度、技术手段、宣传教育、多方协作等）
   *   **语言规范**: 提示使用规范词汇，避免口语化
   *   **针对性**: 强调每条对策要对应材料中的具体问题

6. **确定文件编号和路径**
   *   自动检测 `daily_task_materials/references/` 目录中已有的 `Day{N}_Composite_Measure_Mock.md` 文件
   *   确定下一个编号（如已有 Day1、Day2，则生成 Day3）
   *   如果用户指定了日期或编号，使用指定的编号
   *   文件命名：`Day{N}_Composite_Measure_Mock.md`

7. **生成文件内容**
   *   文件格式严格遵循以下结构：


## 目录结构与命名

*   **输出目录**: `daily_task_materials/references/`
*   **文件命名**: `Day{N}_Composite_Measure_Mock.md`
*   **编号规则**: 自动检测已有文件，递增编号

## 行为准则

*   **不包含答案**: 生成的材料**绝对不能**包含参考答案或参考模板，只提供答题提示
*   **材料真实**: 材料应贴近实际工作场景，问题描述具体、有层次
*   **格式规范**: 严格按照综应A类考试格式，包含明确的作答要求
*   **难度适中**: 材料长度和难度应符合事业编考试标准
*   **主题多样**: 每次生成尽量选择不同的主题，避免重复

## 材料生成检查清单

- [ ] 每个问题是否有具体表现和数据支撑
- [ ] 是否有补充信息（已尝试的措施或面临的困难）

- [ ] 答题提示是否实用

### 语言规范性
- [ ] 材料语言是否符合政府公文风格
- [ ] 问题描述是否具体、有层次
- [ ] 是否避免口语化表达

### 难度控制
- [ ] 材料长度是否在500-800字
- [ ] 问题层次是否清晰，便于梳理
- [ ] 是否符合事业编考试难度标准

## 示例输出

生成的文件应包含：
- 材料部分（背景+4个问题+补充信息）
- 练习任务（题目+作答要求）
- 答题区（问题梳理+对策提炼，空白）
- 答题提示（格式、维度、语言、针对性）

**注意**: 生成的文件**不包含**参考答案或参考模板，用户完成作答后可使用 `/answer_reference` 命令生成答案和点评。
