# 命令：/answer_reference

本规则定义了如何对用户在 Markdown 文件中编写的主观题（如综应A类、面试模拟等）答案进行 **智能点评**，并提供 **标准参考范文**。

## 触发方式
用户输入：`/answer_reference`

## 核心逻辑：智能评估系统 (Intelligent Assessment System)

当用户在练习文件中完成作答后，执行此命令，AI 将扮演 **阅卷老师** 的角色，执行以下评估流程：

1.  **Context Analysis (语境分析)**
    *   **读取材料**: 扫描当前文件中的 **"材料"**、**"背景"** 或 **"题目"** 部分，理解考题要求。
    *   **读取答案**: 识别用户的作答内容（通常位于文件末尾或特定的“作答区域”）。

2.  **Evaluation Strategy (评分策略)**
    *   **采分点匹配**: 根据题目类型（如概括归纳、提出对策、应急处理），检查用户答案是否涵盖核心关键词（Key Points）。
    *   **逻辑结构**: 评估答案的条理性（如是否使用了“一、二、三”或“首先、其次”）。
    *   **语言风格**: 检查是否符合公文写作规范（如是否使用了“法治思维”、“政府话语体系”，还是过于口语化）。

3.  **Output Generation (输出生成)**
    *   **得分预估**: 根据模拟评分标准，给出分数区间（例如：18-22分 / 满分30分）。
    *   **点评反馈**: 明确列出 **"优点 (Pros)"** 和 **"不足/失分点 (Cons)"**。
    *   **标准范文**: 生成一份高质量的参考答案，作为用户的学习标杆。
    *   **改进建议**: 针对用户的具体弱点，给出 1-2 条可执行的改进建议（Actionable Advice）。

## 执行步骤 (Execution Pipeline)

1.  **Append Mode (追加模式)**
    *   不要覆盖用户的内容。
    *   在文件末尾追加内容，使用分割线 `---` 与用户作答区隔开。

2.  **Content Structure (内容结构)**
    请严格遵循以下 Markdown 结构进行输出：

    ```markdown
    ---

    ## 📝 答题点评与参考范文 (Feedback & Reference)

    ### 1. 您的答案点评
    **得分预估**: **[分数]** (满分[总分])

    *   **优点**:
        *   [列出做对的点1]
        *   [列出做对的点2]
    *   **不足 (失分点)**:
        *   **[核心问题]**: [解释] (例如：语言过于口语化)
        *   **[遗漏点]**: [指出漏掉了哪个关键信息]

    ### 2. 参考范文 (Standard Answer)

    **[题目/标题]**

    [这里提供标准的、结构化的、语言规范的参考答案]
    (注意：综应A类范文应包含恰当的小标题和总分结构)

    ### 3. 💡 提升建议 (Action Items)
    *   [针对性的建议1]
    *   [针对性的建议2]
    ```

## 适用场景
*   **综合应用能力 (A/B/C类)**: 案例分析、公文写作。
*   **面试模拟**: 结构化面试题的思路解析。
*   **申论**: 归纳概括、综合分析、大作文提纲。

## 行为准则
*   **客观公正**: 评分要基于材料，不要随意打高分，指出真问题才能帮助用户进步。
*   **语气鼓励**: 在指出问题的同时，肯定用户的努力，保持“亦师亦友”的语调。
*   **格式规范**: 确保生成的 Markdown 格式整洁，重点内容加粗。
