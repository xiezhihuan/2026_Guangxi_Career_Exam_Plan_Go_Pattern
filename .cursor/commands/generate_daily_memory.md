# 命令：/generate_daily_memory

本规则定义了如何生成每日记忆的常识资料，用于公考/事业编学习，支持每日积累知识、按艾宾浩斯记忆曲线复习，并提供助记方法。

## 触发方式

用户输入：`/generate_daily_memory [Day N]` 或 `/generate_daily_memory`

## 核心逻辑：每日记忆资料生成策略 (Daily Memory Generation Strategy)

生成记忆资料时，必须遵循以下原则：

1. **知识点数量**: 固定生成 **3-5个** 知识点，适合每日积累，避免疲劳。每个知识点控制在100-150字。
2. **领域覆盖**: 确保涵盖多个知识领域，包括但不限于：
   *   时政（政策、会议、报告等）
   *   历史（重大事件、人物）
   *   地理（地形、河流、气候）
   *   科技（基础概念、创新应用）
   *   法律（法规要点、原则）
   *   文学/文化（经典作品、传统习俗）
   *   经济（理论、指标）
   *   生物/物理（基础科学知识）
3. **难度适中**: 选择公考常考、易记忆的知识点，既有基础概念也有需要理解的要点。
4. **记忆曲线集成**: 基于艾宾浩斯曲线，提供个性化复习计划（如当天、1天后、3天后、7天后、30天后）。
5. **助记方法**: 每个知识点必须提供1-2种助记技巧（如图像联想、故事法、缩写法），提升记忆效果。
6. **互动性**: 使用 `<details>` 标签隐藏详细解释和助记方法，方便自测和复习。

## 执行步骤 (Execution Pipeline)

1. **确定日期和编号**
   *   如果用户提供了 `Day N`，使用该编号。
   *   如果未提供，自动检测 `daily_task_materials/references/` 目录中已有的 `Day{N}_DailyMemory.md` 文件，确定下一个编号。
   *   如果当前有打开的每日任务文件（如 `2025-12-06_Saturday.md`），从文件名中提取日期信息。

2. **生成文件内容**
   *   文件路径：`daily_task_materials/references/Day{N}_DailyMemory.md`
   *   文件格式严格遵循以下结构：

```markdown

- **学习建议**:

  *   每日结束时，填写"你的记忆笔记"，加深理解
```

3. **知识点质量要求**
   *   每个知识点必须有核心关键词（1-3个），便于记忆。
   *   详细解释要准确、简洁，引用公考权威来源（如报告、教材）。
   *   助记方法要实用、生动，避免生涩；至少1种图像/故事类方法。
   *   分类标签要准确，使用【】标注。
   *   知识点表述要独立完整，便于脱离上下文复习。
   *   确保3-5个知识点覆盖至少3个不同领域。

4. **内容多样性**
   *   优先选择与当前时政热点相关的知识点（如最近政策、节日）。
   *   结合事业编考试大纲，确保实用性。
   *   避免重复之前生成的知识点，逐步构建知识网络。

5. **文件链接更新**（可选）
   *   如果检测到对应的每日任务文件，可以在任务文件的学习时段添加或更新记忆资料文件的链接。
   *   在文件开头添加返回任务卡链接，使用相对路径。

## 目录结构与命名

*   **输出目录**: `daily_task_materials/references/`
*   **文件命名**: `Day{N}_DailyMemory.md`
*   **任务文件**: `daily_tasks/YYYY-MM-DD_Weekday.md`

## 行为准则

*   **幂等性**: 如果目标文件已存在，询问用户是否覆盖，或生成新的编号。
*   **内容质量**: 确保知识点准确无误，提供可靠来源；助记方法原创且有效。
*   **格式规范**: 严格遵循 Markdown 格式，确保在Cursor中渲染良好；使用 `<details>` 实现折叠。
*   **链接检查**: 确保返回任务卡的链接路径正确（使用相对路径）。
*   **个性化**: 鼓励用户在"你的记忆笔记"中填写，增强主动学习。

## 示例输出

生成的文件应包含：
- 清晰的标题、说明和使用指南
- 3-5个知识点卡片（包含关键词、简述、记忆笔记、详细解释、助记方法、复习计划）
- 今日积累总结
- 记忆曲线复习指南和学习建议
