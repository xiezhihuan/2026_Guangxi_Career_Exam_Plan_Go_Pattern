# 命令：/generate_case_analysis

本规则定义了如何生成申论综合分析题（原因+对策类）的评分和点评文档，用于分析用户答案、提供改进建议和标准答案。

## 触发方式

用户输入：`/generate_case_analysis` 或提供原始材料和用户答案

## 核心逻辑：综合分析题评分策略 (Case Analysis Scoring Strategy)

生成评分和点评文档时，必须遵循以下原则：

1. **题型识别**: 自动识别为"综合分析题（原因+对策类）"，这是申论和事业编考试中的常见题型。
2. **评分标准**: 采用事业编考试标准（30分制），包括：
   - 内容要点（18分）：原因分析、影响分析、解决对策
   - 表达规范（8分）：格式规范、语言规范、逻辑清晰
   - 字数要求（2分）：根据题目要求判断
   - 卷面整洁（2分）：假设字迹工整
3. **全面诊断**: 从格式、内容、语言、逻辑四个维度进行全面诊断。
4. **提供标准答案**: 提供两个版本的标准答案（精炼版和规范版），适应不同字数要求。

## 执行步骤 (Execution Pipeline)

1. **解析输入材料**
   *   提取原始材料（Raw Data）：包括标题、来源、正文内容
   *   提取题目要求（Requirement）：问题、字数要求、其他要求
   *   提取用户答案（User Implementation）：用户的作答内容

2. **分析材料结构**
   *   识别现象描述段落
   *   识别原因分析段落（通常包含"原因"、"由于"、"因为"等关键词）
   *   识别影响分析段落（通常包含"导致"、"影响"、"破坏"等关键词）
   *   识别对策建议段落（通常包含"需要"、"应"、"必须"等关键词）
   *   识别总结升华段落（通常包含"唯有"、"只有"、"才能"等关键词）

3. **提取关键信息**
   *   **原因关键词**: "政策空子"、"钻了政策空子"、"蹭穿"、"恶意退货"、"未建立有效的信用识别"、"一刀切"、"降低退货率"等
   *   **对策关键词**: "坚守诚信底线"、"更透明的商品信息"、"构建精细治理体系"、"补充细化规则"、"舆论引导"等
   *   **主体识别**: 消费者、商家、平台、监管部门、社会等

4. **评分和诊断**
   *   **内容完整性检查**: 
     - 原因分析是否完整（通常需要3-4个原因）
     - 影响分析是否充分
     - 解决对策是否覆盖所有主体（通常需要4-5个主体）
   *   **格式规范性检查**:
     - 是否有总起句
     - 序号是否规范（"一是...二是..."）
     - 段落层次是否清晰
   *   **语言规范性检查**:
     - 是否有口语化表达
     - 用词是否准确
     - 句式是否多样
   *   **逻辑性检查**:
     - 原因与对策是否对应
     - 各要点之间是否有逻辑衔接
     - 结尾是否有总结升华

## 6. 🧠 核心考点与思维模型 (Go Pattern)

### Parser 逻辑流 (Pipeline)

将文章看作 `RawData`，你需要编写一个 `Parser` 将其转换为标准的 `AnalysisStruct`。

```go
type AnalysisStruct struct {
    Phenomenon string      // 现象描述
    Causes     []Cause     // 原因分析
    Impact     Impact      // 影响分析
    Solutions  []Solution  // 解决对策
    Summary    string      // 总结升华
}

type Cause struct {
    Level    string // 层面：政策、消费者、平台、商家等
    Content  string // 具体内容
    Evidence string // 材料依据
}

type Solution struct {
    Subject  string // 主体：消费者、商家、平台、监管部门、社会等
    Action   string // 具体措施
    Goal     string // 预期效果
}
```

**Input (段1)**: [现象描述]
**Process A (段X)**: 原因一 —— **[层面]**（[关键词]）
**Process B (段X)**: 原因二 —— **[层面]**（[关键词]）
**Process C (段X)**: 原因三 —— **[层面]**（[关键词]）
**Process D (段X)**: 原因四 —— **[层面]**（[关键词]）
**Process E (段X)**: 影响分析 —— **[影响]**（[关键词]）
**Process F (段X)**: 对策 —— **[多方协同]**（[主体和措施]）
**Output (段X)**: 总结升华 —— **[机制/愿景]**（[关键词]）

### 答题技巧 (Middleware)

#### 1. **结构规范中间件**

**标准格式**：
```
总起句（现象本质）→ 原因分析（3-4点）→ 影响分析（简要）→ 解决对策（4-5点）→ 总结升华
```

**关键要素**：
- ✅ 总起句：用一句话概括现象本质（如："反映了...问题"、"折射出...矛盾"）
- ✅ 原因分析：使用"一是...二是...三是...四是..."的规范表述
- ✅ 对策分析：使用"一是...二是...三是...四是...五是..."的规范表述
- ✅ 总结升华：呼应开头，提出愿景

#### 2. **关键词抓取中间件**

**原因关键词**：
- [从材料中提取的关键词，如"政策空子"、"钻了政策空子"、"蹭穿"、"恶意退货"、"未建立有效的信用识别"、"一刀切"、"降低退货率"等]

**对策关键词**：
- [从材料中提取的关键词，如"坚守诚信底线"、"更透明的商品信息"、"构建精细治理体系"、"补充细化规则"、"舆论引导"等]

#### 3. **语言规范中间件**

**避免口语化**：
- ❌ "[口语化表达]" → ✅ "[规范表达]"
- ❌ "[口语化表达]" → ✅ "[规范表达]"

**使用规范连接词**：
- 原因：首先、其次、再次、最后 / 一是、二是、三是、四是
- 对策：一是、二是、三是、四是、五是
- 总结：唯有...才能... / 只有...才能...

#### 4. **逻辑衔接中间件**

**过渡句示例**：
- 原因与影响之间："这一现象导致了..."
- 影响与对策之间："要解决这一问题，需要..."
- 对策之间："与此同时..." / "此外..." / "更重要的是..."

#### 5. **字数控制中间件**

**分配建议**（500字为例）：
- 总起句：30-40字
- 原因分析：150-180字（每点35-45字）
- 影响分析：40-50字
- 解决对策：200-220字（每点40-45字）
- 总结升华：40-50字

### 备考建议 (Best Practices)

1. **养成规范习惯**：
   - 始终使用"一是...二是..."的规范表述
   - 每个要点前加序号和"是"字
   - 保持段落层次清晰

2. **强化材料意识**：
   - 原因和对策都要从材料中提取，不能主观臆断
   - 注意材料中的关键词和关键句
   - 确保每个要点都有材料依据

3. **提升语言水平**：
   - 多积累规范表达（如"折射"、"凸显"、"破局"等）
   - 避免口语化表达
   - 注意句式变化，避免单一

4. **完善逻辑结构**：
   - 确保原因与对策一一对应
   - 注意各要点之间的逻辑关系
   - 结尾要有总结升华

5. **控制答题时间**：
   - 审题：2-3分钟
   - 阅读材料：5-8分钟
   - 组织答案：5-7分钟
   - 书写：10-15分钟
